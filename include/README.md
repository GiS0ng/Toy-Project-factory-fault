# 프로젝트 구조 및 코드 

## 📂 프로젝트 폴더 구조
```text
└── 프로젝트 루트 폴더
    └── include/
        ├── ErrorCode.h
        ├── IVibrationSensor.h
        └── MachineMonitor.h
```

## 📄 헤더 파일별 요약

### 1. `ErrorCode.h` (상태 코드 정의)
* **타입**: 열거형 데이터 타입 (`enum`)
* **역할**: CSV 포맷 데이터에 적재될 시스템의 상태 코드를 정의합니다.
* **상태 코드 분류**:
  * `NORMAL (0)`: 정상 운전 상태
  * `NEED_INSPECTION (101)`: 육안 점검이 필요한 경고 상태
  * `IMMEDIATE_STOP (102)`: 치명적인 진동 오류가 감지되어 즉시 정지가 필요한 상태

### 2. `IVibrationSensor.h` (센서 인터페이스 및 클래스)
* **`class VibrationSensor`**: 실제 공장 장비에 장착된 물리적인 진동 측정 센서를 프로그램 안에서 하나의 객체(Object)로 다루기 위해 선언한 클래스입니다.
* **`getVibration()`**: 센서의 가장 핵심적인 기능으로, 현재 설비의 실제 진동수 값을 측정하여 반환하는 함수입니다.

### 3. `MachineMonitor.h` (메인 감시 시스템 제어)
* **`VibrationLog`**: 데이터 보관함 양식 구조체로, `[시간]`, `[진동수]`, `[에러코드]`를 한 세트로 묶어 관리합니다.
* **`std::vector<VibrationLog> periodic_buffer`**: 임시 메모리 버퍼입니다. 치명적인 오류 발생 시 시스템이 꺼지기 전, 이 버퍼를 이용해 **"종료 직전까지의 정상 데이터"**를 안전하게 대피 저장합니다.
* **센서 및 내부 함수 선언**: 3개의 센서 객체를 제어하고 이원화 저장(정기 저장, 긴급 대피, 비상 격리)을 수행하는 내부 헬퍼 함수들이 선언되어 있습니다.
* **`runTestScenario()`**: 가상의 진동 데이터를 주입하여 시나리오별로 셧다운 및 데이터 대피가 정상 작동하는지 검증하는 가상 테스트 함수입니다.
